## TDR-001: Agent-First架构

### 背景
传统游戏以人类玩家为中心，AI只是辅助。我们希望探索AI Agent作为核心玩家的可能性。

### 决策
采用Agent-First架构设计：
- 游戏作为平台
- Agent通过API交互
- 开放标准接口

### 原因
1. **研究价值** - 探索AI Agent在游戏中的应用
2. **开放性** - 任何Agent可以接入
3. **灵活性** - Agent可以是LLM、规则系统或人类
4. **人机协作** - Agent与人类共同创造叙事

### 结果
✅ 成功实现Agent管理系统
✅ 提供HTTP/WebSocket API
✅ Agent可完全控制游戏
⚠️ 需要用户安装本地LLM（可选）

## TDR-002: 2D模式选择

### 背景
游戏主题是"人性"，核心是对话和选择，而非复杂空间操作。

### 决策
采用2D模式而非3D：
- 2D场景用于游戏世界
- 对话密集型交互
- Agent感知更简单

### 原因
1. **性能** - 2D渲染更快，适合大量Agent
2. **开发效率** - 2D场景创建更简单
3. **核心体验** - 对话和选择是重点
4. **Agent友好** - 2D感知更直观

### 结果
✅ AgentState2D实现
✅ AgentController2D实现
✅ 性能良好（可支持100+Agent）
✅ 开发效率提高

## TDR-003: 本地LLM方案

### 背景
高质量LLM通常需要外部API，有成本和隐私问题。用户希望完全本地运行。

### 决策
推荐Ollama + Phi-3-mini（2GB）：
- 完全本地、免费
- 质量接近GPT-3.5
- 用户可选安装

### 原因
1. **成本** - 完全免费，无API费用
2. **隐私** - 数据不外传
3. **质量** - Phi-3-mini足够好
4. **大小** - 2GB文件，用户可接受

### 替代方案
- TinyLlama-1.1B（600MB，质量较低）
- llama.cpp（灵活，需手动配置）
- 外部API（高质量，有费用）

### 结果
✅ LocalLLMClient实现
✅ Ollama集成简单
✅ 质量：⭐⭐⭐⭐（接近GPT-3.5）
⚠️ CPU推理：8-15 tokens/s（GPU更快）

## TDR-004: 文字显示系统

### 背景
用户确认本地高质量TTS几乎不可能，应以文字为主。

### 决策
文字显示为主，语音可选增强：
- 打字机效果（25字符/秒）
- 文字格式化
- 系统TTS可选按钮
- 预生成核心对话（可选）

### 原因
1. **可行性** - 文字显示完全本地
2. **体验** - 打字机效果增加沉浸感
3. **性能** - 无延迟，无依赖
4. **成本** - 完全免费

### 结果
✅ DynamicTextDisplay实现
✅ 打字机效果良好
⚠️ 系统TTS质量一般（机器人音）
⚠️ 高质量TTS需外部API

## TDR-005: 语音识别方案

### 背景
用户希望语音控制移动和对话，类似语音助手。

### 决策
Web导出使用Web Speech API，本地可选：
- Web导出：浏览器原生API（推荐）
- 本地：Python脚本或Whisper（可选）

### 原因
1. **Web优势** - 浏览器原生，完全免费
2. **质量** - 中文支持良好
3. **实时** - 响应快（<200ms）
4. **易用** - 用户只需点击按钮

### 本地方案
- Python + SpeechRecognition（免费）
- 本地Whisper（高质量，需安装）
- Whisper API（最高质量，有费用）

### 结果
✅ VoiceInputSystem实现
✅ Web导出自动启用
✅ 支持中文命令
⚠️ 本地导出需要额外配置

## TDR-006: API端口分配

### 背景
Agent需要通过API与游戏交互，需要端口分配。

### 决策
端口分配：
- HTTP API: 8080（REST）
- WebSocket: 8081（实时）
- Ollama: 11434（本地LLM）
- Whisper: 8083（可选）

### 原因
1. **避免冲突** - 不使用常用端口
2. **清晰区分** - 不同服务不同端口
3. **易记** - 简单数字
4. **灵活** - 用户可修改

### 结果
✅ 端口分配清晰
✅ 无冲突问题
✅ 文档标注端口用途

## 决策评估标准

### 评估维度
1. **可行性** - 技术上是否可实现
2. **成本** - 开发成本、运行成本
3. **性能** - 响应速度、资源占用
4. **体验** - 用户体验质量
5. **Agent友好** - Agent使用难度

### 决策流程
1. **识别问题** - 明确需求
2. **提出方案** - 多个候选方案
3. **对比评估** - 按维度对比
4. **选择决策** - 记录选择和原因
5. **实施验证** - 记录结果
